Accessibility Statement

Last assessed: 2026-06-23

Service description

AutoClip (autoclip.studio) is an AI-assisted video editing web application that analyses YouTube channel data to suggest short-form clip candidates. This statement covers the web application and all publicly accessible static pages.

Conformance standard

AutoClip targets conformance with EN 301 549 v3.2.1, which incorporates Web Content Accessibility Guidelines (WCAG) 2.1 Level AA as its operative web accessibility benchmark. EN 301 549 is the harmonised standard under the European Accessibility Act (Directive (EU) 2019/882), enforceable since 28 June 2025.

Conformance status

Partially conforms with WCAG 2.1 Level AA. The application meets many Level AA criteria; known limitations are documented below. We use automated axe-core tooling in CI (Issue 266) to catch critical and serious violations on every pull request.

Known limitations

The following are known areas that do not fully meet WCAG 2.1 Level AA at this time:

  • Complex interactive components — The clip review editor (three-pane layout with video player, transcript, and tool panel) has keyboard navigation patterns that may not be fully operable for keyboard-only users on all browsers. We are tracking this as a remediation item.
  • Colour contrast on muted text — Some secondary/muted text elements use colour combinations that may fall below the 4.5:1 contrast ratio required by WCAG 1.4.3. Primary interactive elements and heading text meet the contrast requirement.
  • Dynamic content announcements — Live task progress notifications (background jobs such as DNA build and clip generation) may not be consistently announced to screen-reader users. ARIA live regions are partially implemented.
  • Video player controls — The embedded video player for clip review does not currently expose full keyboard-accessible controls for all playback operations.

We are committed to addressing these limitations in upcoming releases. Priority is given to the core clip-review workflow.

Assessment approach

AutoClip assesses accessibility through:

  • Automated axe-core testing integrated into the CI pipeline (Issue 266 — runs on every pull request against critical and serious violations).
  • Manual keyboard-navigation testing during development of new interactive components.
  • Periodic review against the WCAG 2.1 checklist.

In the United States, complaints regarding web accessibility for public accommodations may be directed to the U.S. Department of Justice under ADA Title III.
